# Project Overview

## Portfolio Website Planning

### 1. Define Goal and Audience
- Goal: General-use personal portfolio website.
- Major audience:
  - recruiter
  - random website followers (for example from X)
  - friends and coworkers
  - university professor
- One-sentence positioning:
  - `This site should help a broad audience quickly understand who I am, what I build, and why my work is worth following or contacting me about.`

### 2. Recommended Web Page Sections
- About Me
- Academic History
- Work History
- Projects
- Tech Stack
- Contact

### 3. UI Template
- Current template files:
  - `src/index.html`
  - `src/style.css`
  - `src/script.js`
- Template sections:
  - Home / hero with short positioning and CTAs
  - Projects with reusable project cards
  - About with short background and skills
  - Contact with email and profile links
- Replace the placeholders first:
  - `Your Name`
  - `you@example.com`
  - project names, descriptions, tech stacks, impact, and links
  - LinkedIn / GitHub links
- Keep the first version simple: update real content before adding new layout ideas.
